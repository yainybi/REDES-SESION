Sección de preguntas sobre el protocolo HTTP

  • ¿Sobre que protocolo binario está montado el protocolo HTTP?
    Sobre el protocolo TCP.

  • ¿Cuáles son los clientes http y los servidores http mas utilizados?
    Cliente:
    • Navegador
    • CURL
    • Librerias nativas
    Servidores:
    • Apache
    • Microsoft IIS/li>
    • Nginx
    • Google GWS

  • ¿Qué verbos admite un comando en el requerimiento HTTP?
    GET, POST, PUT, DELETE.

  • ¿Qué contenido lleva el body de un requerimiento HTTP?
    Contiene datos ingresados en campos de un formulario y/o archivos enviados y/o subidos al servidor. Además que estos datos pueden estar en cualquier formato.

  • ¿Qué diferencia existe entre un URL y un URI?
    La URI es el recurso como tal, pero URL nos da información para poder encontrar este recurso en Internet, algo así como la dirección postal.
    • URL: (localizador uniforme de recursos).
    • URI: (identificador uniforme de recursos).

  • ¿En qué variables quedan almacenados datos de las respuestas http?
    Quedan almacenados sobre variables denominadas cookies.

  • ¿Qué significa Virtual hosting?
    Es una de la mejoras que tuvo HTTP1.1 que no tenia HTTP1.0, que trata sobre múltiples dominios sobre una misma ip, es decir que en un mismo servidor web se maneje múltiples nombre de hosting distinto sobre esa misma ip.

  • ¿Por qué el HTTP se considera STATE LESS?
    Significa que es un protocolo sin estado. No puede mantener continuamente el estado o sesión de una aplicación, es decir, no guarda ningún dato entre dos peticiones en la mísma sesión, ya que después de un determinado número de mensajes de requerimiento y respuesta, la conexión se rompe, por diferentes razones, puede ser por parte del cliente que no siguió enviando requerimientos, o por parte del servidor, que por un determinado tiempo por no recibir un requerimiento rompe la conexión.
